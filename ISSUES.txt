
Open items in Eclipse Common Lisp


1.  K&R (old-style) function definitions throughout C source.

    The C source uses pre-ANSI K&R function definition syntax
    throughout interface.c and eclipse.c (hundreds of functions).
    These are suppressed with -Wno-old-style-definition in the
    makefile.  While not a correctness issue, K&R definitions prevent
    the compiler from checking argument types at call sites.

2.  No FASL format or image save/load.

    The byte-code compiler was never completed (skeleton in
    lisp/non-build/byte.lisp).  There is no way to save a Lisp image
    or produce fast-load files.  The only compilation path is
    compile-file to C.

3.  No thread support.

    A threads module exists (lisp/non-build/threads.lisp) but is not
    part of the build.  The runtime is single-threaded.

4.  ed function not implemented.

    The standard ed function does nothing.

5.  Fixnum range is 29 bits.

    MOST-POSITIVE-FIXNUM is 536,870,911 (2^29 - 1).  This is smaller
    than most modern implementations provide.

6.  Float type conflation.

    short-float and single-float are the same type.  double-float and
    long-float are the same type.  This is permitted by the ANSI
    standard but may surprise users expecting four distinct float types.
